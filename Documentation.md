# ZAD PROJECT — AGENT CONTEXT

## SYSTEM OVERVIEW

Zad is an **AI-powered Islamic Educational Q&A System**.

Users interact through a Flutter mobile app where they can ask Islamic questions (Quran, Hadith, Fiqh, etc.), and receive accurate, cited answers generated by an external AI system.

### Core Architecture Philosophy

- Backend does NOT perform AI reasoning
- Backend acts ONLY as an orchestrator
- External AI (RAG service via HTTP) is responsible for:
  - Understanding questions
  - Retrieving knowledge
  - Generating answers

The backend is responsible for:
- Authentication (JWT)
- Managing users and roles
- Managing chat sessions and history
- Building structured prompts
- Sending requests to external AI
- Receiving and storing responses
- Returning responses to client

## ARCHITECTURE EXPLANATION

### Domain Layer (Core - PURE)
- Contains Entities, Enums, Value Objects
- Contains business rules
- NO dependencies
- NO EF Core
- Pure C# only

---

### Application Layer
- Contains DTOs, Services, Interfaces, Validators
- Handles business logic and workflows
- Depends ONLY on Domain
- Does NOT access database directly

---

### Infrastructure Layer
- Implements database access (EF Core)
- Contains DbContext
- Implements repositories
- Implements Unit of Work
- Contains external AI HTTP client
- Depends on Domain + Application

---

### API Layer
- Controllers (REST endpoints)
- Middleware (Auth, Error Handling)
- JWT configuration
- Swagger setup
- Depends on all layers

---

## DATA FLOW

1. Mobile app sends HTTP request (with JWT)
2. API Controller receives request
3. JWT is validated
4. Request is passed to Application Service
5. Service builds AI prompt
6. Service calls external AI via HTTP client
7. AI returns answer + citations
8. Backend stores response in database
9. Response returned to client
